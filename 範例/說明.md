# 範例客戶：Sean 🏀（@still.hooping.sean）

> 這份文件幫你做一件事：讓你知道元件庫每個位置該放什麼、長什麼樣。

這是一個**完整填好的元件庫範例**，用真實帳號「31 歲的籃球夢」示範。品牌檔案部訪談時，問到哪一項就把對應的檔案秀給客戶看：

| 訪談問到…… | 秀這份 |
|---|---|
| 你是誰、給誰看、想帶來什麼 | [我是誰.md](./我是誰.md) |
| 你怎麼說話（口氣萃取長怎樣） | [我怎麼說話.md](./我怎麼說話.md) |
| 版面截圖要截什麼 | [IG截圖/](./IG截圖/) |
| 舊貼文、轉錄稿存起來長怎樣 | [舊貼文/](./舊貼文/)（內含 Reels 轉錄稿範例） |
| 對標（別人的內容）存起來長怎樣 | [對標/](./對標/)（見資料夾內說明） |
| 做好的成品放哪 | [成品/](./成品/)（依平台分資料夾） |

提醒：這是「範例」不是「模板」——客戶自己的元件庫建在專案根目錄的 `我的品牌/`（已被 `.gitignore` 擋住，不會進 git），內容照他的品牌來，不要抄這裡的。**客戶的任何真實資料都不准存進這個 `範例/` 資料夾**，這裡只放示範用的內容。

**鐵則：這個資料夾裡只放真實存在的內容**（本人真的發過、真的轉錄過的）。缺範例就留說明檔，不准虛構一份充數——虛構的「範例」會讓客戶以為造假是正常流程。
