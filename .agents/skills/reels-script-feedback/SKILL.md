# 短影音腳本回饋：把客戶的習慣存下來

每次短影音腳本交付（定稿或客戶給完修改意見）後執行，不用等客戶要求。

## 存哪裡

`我的品牌/腳本偏好.md`（客戶元件庫，已 gitignore，不會被 commit）。檔案不存在就建立；存在就**追加更新，不要整份覆蓋**。

## 記什麼（只記觀察得到的，不腦補）

依 reels-script-framework 的五段整理：

- **Hook**：客戶偏好哪種鉤子（痛點？反差？數字？），有沒有他改掉過的開場方式。
- **Build-Up**：他喜歡多短，有沒有嫌過「鋪太長」。
- **Core Value Bomb**：偏好幾點式還是敘事式、每點多長。
- **Proof or Payoff**：他習慣的自介句式（例如「我是阿閎，也是 POVU 的工程師」）。
- **CTA**：固定的關鍵字句式、有沒有招牌收尾（例如小聲補一句「完全免費」）。
- **用詞**：這次被客戶改掉的字 → 同步問一句要不要更新 `我怎麼說話.md` 的禁忌詞。

每條記錄註明日期與來源（哪一支腳本、客戶原話），像這樣：

```markdown
## CTA
- 2026-07-15（AI 剪片新手包）：CTA 固定用「留言『關鍵字』我就傳給你」，結尾小聲補一句降低推銷感。
```

## 流程

1. 讀現有的 `我的品牌/腳本偏好.md`（沒有就跳過）。
2. 對照這次的初稿 vs 定稿，列出客戶實際改了什麼、說了什麼。
3. 把新觀察追加進對應段落，唸給客戶聽一次確認「我記下這些習慣，下次直接套用，對嗎？」。
4. 客戶確認後存檔。之後每次寫腳本，reels-script-framework 動筆前**連同 我怎麼說話.md 一起讀這份檔案**。
