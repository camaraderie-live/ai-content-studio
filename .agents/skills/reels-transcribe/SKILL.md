# Reels 下載＋轉錄

整條鏈：yt-dlp 下載 → ffmpeg 抽音訊 → ElevenLabs 轉錄 → 簡轉繁 → 問一題 → 存檔。全程在客戶電腦本機完成，不用付費服務（ElevenLabs 用既有的免費額度金鑰）。

## 執行步驟

工作檔放在暫存資料夾，轉錄稿才存進元件庫。

```bash
# 1. 下載（公開貼文不需要登入）
yt-dlp -o "reel.%(ext)s" "<REELS_URL>"

# 2. 抽音訊（轉錄不需要畫面，音檔小很多）
ffmpeg -y -i reel.* -vn -ac 1 -ar 16000 audio.mp3

# 3. ElevenLabs 轉錄（金鑰沿用環境檢查時設定的那把）
curl -s -X POST https://api.elevenlabs.io/v1/speech-to-text \
  -H "xi-api-key: $(sed -n 's/^ELEVENLABS_API_KEY=//p' ~/.codex/skills/video-use/.env)" \
  -F model_id=scribe_v1 \
  -F file=@audio.mp3
```

回傳 JSON 的 `text` 欄位就是逐字稿。

## 簡轉繁（必做）

ElevenLabs 轉錄中文常輸出**簡體**。存檔前你自己把全文轉成**繁體中文**（你本來就會，不用裝額外工具），順手把明顯的錯字、斷句修順，但**不改原意、不改口語詞**。存進元件庫的一律繁體。

## 轉錄完：問一題再存檔（不要自己猜）

**「這支影片是別人的、拿來參考學習的？還是你自己以前發過的？」**

- 別人的 → 存 `<元件庫>/對標/`，檔名 `Reels轉錄-<主題或帳號>-<日期>.md`
- 自己的 → 存 `<元件庫>/舊貼文/`，同樣命名（檔名除了 Reels、IG 這種平台名，一律用中文）

檔案開頭記上來源網址和日期。存進 `對標/` 的，在檔案開頭加一行提醒：

> ⚠️ 這是別人的內容，僅供學習「結構與節奏」，生成時不可原文照抄任何句子。

跟客戶也白話講一次：「這種對標腳本我們只學它怎麼鋪陳，句子不能抄，抄了會有著作權問題。」

## IG 擋下載時的備援

下載失敗且錯誤訊息像被 IG 擋（rate limit、login required），改用客戶瀏覽器的登入狀態重試：

```bash
yt-dlp --cookies-from-browser chrome -o "reel.%(ext)s" "<REELS_URL>"
```

（客戶用 Safari 就換 `safari`。）跟客戶白話解釋：「IG 有時候會擋沒登入的下載，我借用你瀏覽器裡已經登入 IG 的身分再試一次，不會動到你的帳號。」

## 失敗情境白話說明（不要丟原始錯誤訊息）

| 情況 | 跟客戶怎麼說 |
|---|---|
| 私人帳號 | 「這個帳號是不公開的，我拿不到影片。如果你自己追蹤了這個帳號，我可以借用你瀏覽器的登入身分試試看。」 |
| 影片被刪／網址錯 | 「這支影片好像已經被刪掉了，或網址不完整。麻煩再確認一下連結。」 |
| 用了 cookies 還是被擋 | 「IG 這邊暫時把下載擋住了，通常過幾個小時就會恢復。或者你也可以用手機把影片存下來直接丟給我，我一樣能轉錄。」 |
| 轉錄回傳 401 | 「轉錄鑰匙好像過期了，我們重新設定一次。」（走環境檢查的金鑰流程） |

同一支影片重試不超過 2 次，失敗就走上表說明，不要無限重試。
