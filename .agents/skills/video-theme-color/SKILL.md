# 影片主題色（品牌色）

管理一支影片裡所有疊加文字、字幕外框、動畫強調的顏色，讓整支看起來是同一套設計。

## 兩個「永遠要先問」的鐵則（load-bearing）

**永遠不要自己決定主題色，也永遠不要自己決定套用範圍。這兩件事都要跟使用者確認過才動手。**

1. **主題色是什麼** —— 顏色本身（hex）。
2. **要套用在哪些元素** —— 字幕重點字？依照現有動畫逐項問，不要「全部套上」也不要「只套一個」憑自己猜。

> 為什麼寫成鐵則：曾經只問了「用 Airbnb 的顏色」就把主色套到所有元素，沒問使用者想套哪些，被明確糾正。**確認範圍的成本遠低於整支重上色重渲染。** 就算使用者只丟一句「用 X 品牌的色」，也要回問一次套用範圍再動工。

## 第一步：拿到主題色

使用者可以用三種方式給你，對應處理：

1. **直接講顏色**（「用我們品牌的橘色 #ff6b00」「深藍配白」）—— 有 hex 最準；只給形容詞就先抓一個接近的 hex 給他確認。
2. **丟截圖 / logo** —— 從圖裡吸出主色 + 幾個輔助色，整理成色票（主色 + 文字色 + 背景色）給他確認。
3. **不知道要用什麼** —— **不要叫他自己去看 GitHub**。用 `mcp__visualize__show_widget` 畫色塊卡片（每塊標顏色 + hex），把下面 4 個知名品牌方向擺出來讓他挑：

   | 品牌 | 主色 | 個性 |
   |------|------|------|
   | Airbnb | `#ff385c` 珊瑚紅 | 溫暖、親切、生活感 |
   | Apple | `#0066cc` 科技藍 | 乾淨、極簡、高級 |
   | BMW | `#1c69d4` 沉穩藍 | 專業、企業、可信賴 |
   | Codex | `#cc785c` 陶土橘 | 柔和、溫潤、人文 |

   來源：開源專案 [awesome-design-md](https://github.com/VoltAgent/awesome-design-md)（上百個知名品牌官方配色）。使用者不用點網址，你把色票畫成圖給他看就好。要清單外的品牌，去該 repo 的 `design-md/<品牌>/DESIGN.md` 抓 `colors:` 的 hex。

## 第二步：確認套用範圍（逐項問）

**先看這支影片實際做了哪些疊層 / 動畫**（看它的 EDL `overlays`、`edit/animations/` 底下有哪些 composition），把「這支影片真的有的元素」列成選項，用 `AskUserQuestion`（multiSelect）讓使用者勾。**不要照搬一份固定清單**——每支影片的元素都不一樣，前一支有的（例如數字 counter、終端機視窗、某個裝飾）這支不一定有。沒勾的維持原樣或用白色。

要列選項時，用白話描述你看到的元素讓使用者對得上，例如「字幕裡的重點詞」「某個數字滾動動畫」「某段疊上去的文字」，一項一項照這支影片實際內容列。

「語意固定」的顏色不要換也不要列進去問：macOS 視窗紅黃綠三顆鈕、被打叉的紅色 ✕、終端機慣例的綠色等，這些換了會讓人看不懂。

## 第三步：記錄到 theme.md（**專案本地，每支影片各自一份**）

**主題色是「這支影片專案」的設定，每支影片各自一個主題色，所以 theme.md 一定放在該影片專案資料夾本地 `<videos_dir>/theme.md`，絕對不要放全域**（全域會讓所有影片共用同一色，是錯的）。

```markdown
# 主題色 — 專案：<影片專案名>｜方向：<品牌>
- primary（主色，重點文字／動畫強調）: #ff385c
- ink（主要文字）: #1a1a1a
- canvas（背景／字幕底）: #ffffff
- on-primary（壓在主色上的文字）: #ffffff
- accent（次要點綴，可省略）: #ff5a5f
# 套用範圍: <照這支影片實際勾選的元素填，例如「字幕重點字」，不要照抄別支影片的清單>
```

**每次開場前先讀一次「這個影片專案」的 `<videos_dir>/theme.md`**：已有主題色就跟使用者說「我沿用這支影片之前定的主題色（例如珊瑚紅 #ff385c，套在你當時勾的那些元素），要換再說」，不要每次從零問；沒有 theme.md（＝新影片）就照上面流程主動問。換影片＝換專案資料夾＝讀那份專案自己的 theme.md。

## 第四步：套用（技術實作）

把主色集中成單一色值，方便一鍵換色重渲：

- HyperFrames 疊層的顏色寫在該專案自己的生成器 / composition 檔裡，換色時對那個檔做 `sed 's/#舊色/#新色/g'`，重跑生成 → 重渲兩層 webm → ffmpeg 合成，不要手改 render 好的成品。（生成器檔名依各專案而定，不要假設叫某個固定名字。）
- 換色只動「使用者勾選的元素」對應的 CSS，沒勾的別碰。
- 換完抽 1~2 格（字幕重點字那格 + 數據動畫那格）確認顏色對、對比夠（珊瑚紅／深色主色在白底上要有黑描邊才讀得清楚）。

## 對比安全

主色壓在淺色背景（白牆、白衣）上時，重點字要有描邊（`-webkit-text-stroke` 黑色）或陰影，否則糊掉。深色主色配白底、亮色主色配深底。換色後務必實際抽格看，不要假設。
