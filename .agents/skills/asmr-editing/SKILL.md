# ASMR／無對白素材：用波形找「聲音最明顯的段落」剪，不要平均縮短每段

> **套用前提（gate）**：使用者明講要 ASMR／音效導向，或你先確認過他要這種剪法、他說好。無對白素材還有別的剪法（配旁白、配樂快剪、montage）——先確認，見 `aroll-guide` 情況 C。

當素材沒有對話（例如開箱、包裝、手作過程這類 ASMR 或音效導向的影片），video-use SKILL.md 裡「音檔優先於畫面」的原則要換一種用法：不是靠逐字稿找語句邊界，而是靠**波形圖**找每支素材聲音最密集、最明顯的段落。

1. **不要轉錄。** 沒有對話就不需要 `transcribe_batch.py` / `takes_packed.md`，直接跳到用 `timeline_view.py` 看波形+畫面。
2. **逐支素材看波形，手動標出聲音尖峰區段**，而不是把每支素材都機械式地縮短成同樣比例或同樣秒數。連續型音效（摩擦、攪拌、揉紙）幾乎整段都有聲音，可以整段保留；單一音效（開合、撕膠帶、放下物品）只取尖峰前後一小段。
3. **開始剪之前，先確認素材順序，不要自己假設**，不要因為音量大小重新排序。依序問清楚：
   1. **先看檔名，確認目前的命名方式**——檔名裡有沒有看得出來的編號或順序（例如 No.1、No.2…）。
   2. **如果檔名有順序**：問使用者「要不要照這個檔名順序剪」。使用者說不要，才改問「要不要改用檔案的時間戳排序」。
   3. **如果檔名看不出順序**：先問使用者「要不要我先幫你調整一下檔名順序，或是改用檔案的時間戳排序」。
   4. **如果上面都不要**：問使用者「要我幫你決定順序，還是你提供一份順序清單給我」。
   一旦順序確認下來就照做，不要中途又用音量或畫面內容重新洗牌。
4. **抓總長度時留一點餘裕往上抓，不要一開始就往下修剪過頭**——先估算「全部素材原始總長」與「目標長度」的比例，再抓每段大概該留多少，抓完通常會比目標秒數略短一點，這是正常的（例如目標 35 秒、實際落在 32~33 秒都算合理誤差），不用硬湊到剛好那一秒。
5. 交界處一樣要套用 video-use SKILL.md Hard Rule 3 的 30ms 音訊淡入淡出——ASMR 素材的爆音問題比一般影片更明顯，剪完務必用 `timeline_view.py` 對著**輸出檔**的每個交界點抽查一次波形，確認沒有斷音或爆音。
