# 短影音腳本固定框架

每一支短影音腳本，**固定照這五段的順序寫，缺一段就不算完成**：

| 段落 | 要做到什麼 | 常見失敗 |
|---|---|---|
| 1. Hook（鉤子） | 前 3 秒抓住人：痛點、反差、爭議句、具體數字 | 自我介紹開場、鋪陳背景 |
| 2. The Build-Up（鋪陳） | 一兩句把賭注講清楚：看下去能得到什麼 | 鋪太長，觀眾還沒到重點就滑走 |
| 3. The Core Value Bomb（核心價值） | 本體。具體、可執行、有編號的乾貨；這段占全片最大篇幅 | 空泛口號、聽完不知道怎麼做 |
| 4. The Proof or Payoff（證明） | 為什麼信你：身分、實績、或眼前這支影片本身就是證據 | 放在開頭變成自我介紹 |
| 5. The Clear CTA（行動呼籲） | 一個明確動作：留言關鍵字、追蹤、私訊 | 一次叫觀眾做三件事 |

## 輸出格式

- 逐段標明段落名（Hook: …、The Build-Up: …），讓客戶一眼看出結構。
- 其餘規則照 content-planner 型態三：全程口語、每句一個呼吸的長度、括號動畫標註照 [腳本動畫標註.md](../../../腳本動畫標註.md)、總長 30~60 秒。
- 動筆前一樣要過 profile-library 守門（讀 我是誰.md、我怎麼說話.md）。

## 範例（完整五段）

> **Hook:** AI 剪片效果也太差了吧！根本浪費我的時間
>
> **The Build-Up:** 如果你想使用 AI 幫你剪片，掌握好三個重點，成品一定不會太差
>
> **The Core Value Bomb:** 你如果有在剪影片，你一定試過很多 app 裡面的 AI 剪片、一鍵生成這種功能，很多甚至還要花錢，結果出來的效果（露出無言的表情）
> 1. 先錄旁白或是口播，如果你不打算講話也可以跳過，這個是 AI 判斷的第一個依據
> 2. 好的素材很重要，就連檔名對 AI 來說也很重要，AI 是「讀」你的影片，不是像人類一樣「看」影片（把看放大），他是透過軟體把每一個畫面擷取出來，去認識你的影片，所以檔名對他來說非常有幫助，也就是說對你的成品也很有幫助
> 3. 素材不要太長，AI 說實在並不知道你到底想要素材的哪一個片段，我測試過非常多遍，最有效的方法是你在一開始就把素材整理好，盡量只保留你要的片段，一段影片最好是 5 秒內最能吸引觀眾的注意力
>
> **The Proof or Payoff:** 我是阿閎，也是 POVU 的工程師，你看到的這個影片完全是 Claude 幫我剪的
>
> **The Clear CTA:** 以上這些我都已經整理在我的 AI 剪片新手包，留言「給我新手包」（放大給我新手包）我就傳連結給你（小聲的說：完全免費、沒有要賣課）

## 寫完之後（必做）

腳本初稿完成 → 走品管 `quality-check` → 交付後**觸發 `reels-script-feedback`**，把客戶這次的修改意見與習慣存進他的元件庫，下次越寫越像他。
