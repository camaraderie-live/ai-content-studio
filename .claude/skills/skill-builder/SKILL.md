# skill-builder：幫客戶建自己的 skill

跟客戶解釋 skill 是什麼時用白話：「就是把你常用的做法寫成一張 SOP 卡片，以後說一句話我就照做，不用每次重講。」

## 硬性流程（四步，不可跳）

### 1. 一次一題問清楚

- 這個 skill 要幫你做什麼？（一句話）
- 什麼時候該自動啟動？（客戶說哪些話、什麼情境）
- 有沒有固定的步驟或規則？（照他描述整理，唸回去確認）

### 2. 查重（必做，先查再建）

掃 `.claude/skills/` 底下**全部** skill 的 `name` 和 `description`：

```bash
head -5 .claude/skills/*/SKILL.md
```

跟新需求重疊 → 跟客戶說「你要的跟現有的『某某』很接近，我建議把它擴充就好，不用另建一個」。**客戶明確說還是要新建，才新建。**

### 3. 生成並存檔

- 存到 `.claude/skills/my-<名字>/SKILL.md`，**一律 `my-` 前綴**——跟官方 skill 隔開，之後 `git pull` 更新專案不會動到它。
- 格式照本資料夾其他 SKILL.md：frontmatter 的 `description` 要寫清楚「什麼時候套用」，內文用客戶自己的話寫規則。

### 4. 真實例子實測

拿客戶一個真實的案例照新 skill 跑一遍給他看。**客戶說滿意，才算建好**；不滿意就改，改完再測。

## 建好之後

問一句：「要不要在你的元件庫記一份『我的 skill 清單』？」要的話在 `<元件庫>/我是誰.md` 加一節，列出 skill 名稱＋一句話用途，之後訪談或交接都查得到。
