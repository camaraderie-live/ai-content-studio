# 品管部：交付前三關

任何交付物在給客戶看「完成版」之前，先走完這三關。**沒過就退回給產出它的部門修改，修完重新過品管，不要帶病交付。**

## 第一關：像不像本人（抓 AI 味）

打開客戶元件庫的 `voice.md`，逐項對照：

- 禁忌詞有沒有出現？（voice.md 列的絕不用的詞）
- 句式像不像？（客戶習慣短句就不能出現三行長句）
- **AI 味檢查清單**——出現任何一條就退回：
  - 空泛開場：「在這個時代」「眾所周知」「隨著……的發展」
  - 條列式湊字：沒有觀點、只有「三個技巧、五個方法」的空殼
  - 過度工整：每段字數幾乎一樣、排比句連發
  - 假熱情：「太棒了！」「趕快收藏起來！」這種客戶本人不會說的話
  - 學了對標的「句子」而不是「結構」（對照 benchmarks 原文抽查）

## 第二關：平台格式

| 平台 | 檢查項 |
|---|---|
| FB 長文 | 首行是鉤子；段落 1~3 行；hashtag ≤3；結尾有互動句 |
| IG 輪播 | 6~10 張；每張一重點、字數 ≤30；封面只有一句；末張 CTA |
| IG Reels 腳本 | 前 3 秒鉤子；口語；含動畫標註；150~300 字 |
| Threads | ≤500 字；首句成立；無 hashtag |
| 小紅書 | 標題帶關鍵字＋情緒；正文分段帶 emoji 小標；hashtag 5~10 個 |

## 第三關：影片自檢（只有影片交付時）

沿用 video-use SKILL.md 第 7 步的自我檢查，重點抽查：

- 字幕有沒有被其他疊層蓋住（字幕永遠是 `overlays` 最後一個）
- 主題色是否照 `<videos_dir>/theme.md`、對比夠不夠
- 開頭結尾有沒有黑屏、斷音
- 實際抽 2~3 格畫面看，不要只看指令跑成功

## 結果處理

- **不過** → 列出具體哪一條沒過、退回修改；修完重跑品管。同一稿退回超過 2 次，停下來跟客戶確認方向是不是一開始就歪了。
- **過** → 交付給客戶，存 `成品/<平台>/`，並問「這次的成果要不要存進元件庫？」（走 `profile-library`）。
